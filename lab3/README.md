# Лабораторная работа №3: Наследование и полиморфизм

## Цель работы
Освоить механизм наследования классов и полиморфизм.

## Классы

### Server (базовый)
- Атрибуты: name, ip, cpu, ram, status, connections
- Методы: start(), stop(), add_connection(), get_score()

### WebServer (дочерний)
- Добавлено: domain, sites
- Новый метод: add_site()
- Переопределен: get_score(), __str__()

### DatabaseServer (дочерний)
- Добавлено: db_type, databases
- Новый метод: create_db()
- Переопределен: get_score(), __str__()

## Связь с ЛР-1 и ЛР-2
- Server из ЛР-1
- ServerCollection из ЛР-2

### Сценарий 1: Разные типы в одной коллекции
- Создаются объекты WebServer и DatabaseServer
- Проверка типов через isinstance()
- Объекты добавляются в одну коллекцию
- Выводится список всех серверов

### Сценарий 2: Полиморфизм
- У всех объектов вызывается метод get_score()
- Базовый метод (cpu * ram) дает разные числа
- Переопределенный метод добавляет бонусы (за сайты и БД)
- Видно, что один метод работает по-разному для разных типов

### Сценарий 3: Фильтрация и управление
- Отбор только веб-серверов из коллекции
- Отбор только БД-серверов из коллекции
- Остановка и запуск всех серверов
- Добавление подключений к серверам
